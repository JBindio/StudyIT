1과목 1-3
1.요구사항 분석
	1)현행 시스템 파악
		- 1단계 : 구성, 기능, 인터페이스
		- 2단계 : 아키텍쳐(구조), 소프트웨어
		- 3단계 : 하드웨어, 네트워크
	2)플랫폼 기능 분석
		- 싱글 사이드 플랫폼 : 본인
		- 투 사이드 플랫폼 : 중개
		- 멀티 사이드 플랫폼 : 그룹
	3)운영체제 분석
		- 시스템 소프트웨어 
		- 응용 소프트웨어
		- 인터페이스
	4)네트워크 분석
		- 노드
		- 프로토콜 : 통신규칙(약속)
			- 구문, 의미, 타이밍 ★
	5)DBMS(데이터베이스) 분석
	6)미들웨어 분석 : 중간 매개 역할
		- RPC : 프로시저 원격호출
		- MOM : 메세지
		- ORB : 객체지향 전송
		- DB접속 미들웨어 : APP - DB 연결
		- TP모니터 : 트랜잭션 관리
		- WAS : 웹 어플리케이션 서버
2.요구 공학
	1)고객의 요구를 체계적으로 수집, 분석, 명세화, 검증
	2)요구사항의 분류
		- 기능적 요구사항 : 소프트웨어를 구성하는 기능
		- 비기능적 요구사항 : 보안, 성능, 품질, 안정성
	3)요구사항 개발 프로세스
		- 도출 : 인터뷰, 설문조사 등등
		- 분석 : 분류, 모델링, 설계
		- 명세화 : 시스템 정의서, 명세서
		- 확인 : 검토, 프토타이핑, 인수 테스트
	4)요구사항 분석 CASE 도구
		- 상위 CASE : 계획, 분석, 설계
		- 하위 CASE : 구현, 문서화, 테스트
		- 통합 CASE : 상위, 하위 통합
	5)HIPO 도구
		- 하향식 개발 문서화 도구
		- 가시적 도표 : 큰그림
		- 총체적 도표 : 기능 (입력,출력,처리)
		- 세부적 도표 : 세분화
